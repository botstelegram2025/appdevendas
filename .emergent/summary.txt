**original_problem_statement:**
The user wants to create a mobile application (Android and iOS) for selling digital products (e.g., app activations, IPTV credits), based on an existing Telegram bot's flow.

**PRODUCT REQUIREMENTS:**
1.  **Customer Area:**
    *   User registration/login with phone number validation.
    *   Product catalog with categories and images.
    *   Shopping cart.
    *   Checkout via Mercado Pago (PIX).
    *   Order history screen where users can track payment and delivery status, and view order details.
    *   Ability to edit profile information.
    *   Automatic logout after a period of inactivity.
2.  **Admin Area:**
    *   Separate admin login.
    *   Dashboard with sales reports (monthly revenue, comparisons, charts).
    *   CRUD management for products, categories, and orders.
    *   Admin should be able to view all order details, easily copy them, and confirm order delivery.
3.  **Notifications:**
    *   WhatsApp notifications for both customer and admin for new orders, payment approvals, and deliveries, using an external WAHA (WhatsApp HTTP API) service.
    *   The admin notification for a new order must include all custom fields filled by the user (MAC, OTP, etc.).
    *   The admin should connect their WhatsApp by scanning a QR code in the admin panel. The connection should be resilient to disconnections.
4.  **Deployment/Access:**
    *   The user wants a direct URL to access the application in a web browser (PWA).

**User's preferred language**: Portuguese (português). INSTRUCT the next agent to respond in Portuguese.
